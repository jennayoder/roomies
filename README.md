# Roomies — Roommate Household Tracker

A Flutter app for managing shared household responsibilities: rent tracking, expense splitting, chore scheduling, and event planning.

## Tech Stack

- **Flutter** (Web, Android, iOS)
- **Firebase Auth** — Email/password authentication
- **Cloud Firestore** — Real-time NoSQL database
- **Firebase Hosting** — Web deployment

---

## Prerequisites

- [Flutter SDK](https://flutter.dev/docs/get-started/install) ≥ 3.1.0
- [Firebase CLI](https://firebase.google.com/docs/cli) — `npm install -g firebase-tools`
- [FlutterFire CLI](https://firebase.flutter.dev/docs/cli) — `dart pub global activate flutterfire_cli`
- A Firebase project (see below)

---

## 1. Create a Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com)
2. Click **Add project** → name it `roomies` (or similar)
3. Enable **Google Analytics** (optional)
4. In the project dashboard, enable the following:
   - **Authentication** → Sign-in method → **Email/Password**
   - **Firestore Database** → Start in **production mode** (update rules after setup)
   - **Hosting** (for web deployment)

---

## 2. Configure FlutterFire

From the project root, run:

```bash
# Login to Firebase
firebase login

# Configure FlutterFire — this generates lib/firebase_options.dart
flutterfire configure --project=YOUR_FIREBASE_PROJECT_ID
```

Select **web**, **android**, and **ios** platforms when prompted.

This will create/update:
- `lib/firebase_options.dart`
- `android/app/google-services.json`
- `ios/Runner/GoogleService-Info.plist`
- `firebase.json`

---

## 3. Firestore Security Rules

In the Firebase Console under **Firestore → Rules**, use these starter rules:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users can read/write their own profile
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }

    // Household members can read/write household data
    match /households/{householdId} {
      allow read, write: if request.auth != null &&
        request.auth.uid in resource.data.memberIds;

      match /{subcollection}/{docId} {
        allow read, write: if request.auth != null &&
          request.auth.uid in get(/databases/$(database)/documents/households/$(householdId)).data.memberIds;
      }
    }
  }
}
```

```bash

```


## 6. Deploy to Firebase Hosting (Web)

```bash

flutter build web --release

# Deploy to Firebase Hosting
firebase deploy --only hosting
```

## Household Flow

1. **Register** → creates a User doc in Firestore
2. **Create household** → generates an invite code, saves Household doc
3. **Join household** → enter invite code → added to memberIds
4. All sub-collections (rent, expenses, chores, events) live under `/households/{id}/`
